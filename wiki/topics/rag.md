# RAG

## 主题定义

Retrieval-Augmented Generation（RAG）指在生成前或生成过程中引入外部知识检索，以提升答案的事实性、时效性和可追溯性。

## 为什么重要

- 它是本仓库所有“知识问答 / 研究分析 / 报告生成”系统的底座能力。
- 它把 LLM 从“只依赖训练参数”转为“可以利用外部证据”的系统。
- 后续的 [[graph-rag]]、[[agentic-rag]]、知识图谱增强方案都可以看作 RAG 的扩展分支。

## 本仓库中的设计空间

- 传统 RAG：向量或 BM25 检索后直接生成
- 知识增强 RAG：把知识图谱或结构化知识作为检索基底
- Corrective RAG：在生成前后做文档评分、查询重写、补充检索、幻觉检查
- Agentic RAG：把检索放入更长的 Agent 执行循环，由模型动态规划多步检索与工具调用

## 当前共识

- 单次“检索 + 生成”适合简单问答，但对复杂推理、跨文档综合和长任务不够稳。
- 结构化知识和执行循环都能显著提升复杂问题的处理能力。
- 工程上，RAG 不只是模型问题，还涉及状态、上下文、工具治理与评估。
